# Final Mock Round

## Round Format

1. Explain gRPC in two minutes.
2. Design a payment API with idempotency.
3. Evolve a proto safely.
4. Debug `DEADLINE_EXCEEDED`.
5. Secure the service with mTLS and authz.
6. Deploy behind Envoy in Kubernetes.
7. Define metrics, traces, logs, and SLOs.

## Scoring

| Level | Signal |
|---|---|
| beginner | explains proto/stub/RPC basics |
| intermediate | handles statuses, deadlines, grpcurl, testing |
| senior | covers evolution, resilience, security, observability |
| staff | covers rollout safety, governance, incident prevention |

## Pass Condition

You pass when your answer includes tradeoffs and failure modes without being prompted.
