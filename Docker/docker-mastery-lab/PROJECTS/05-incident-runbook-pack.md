# Project 05: Docker Incident Runbook Pack

## Outcome

Create a runbook set that can guide a teammate during a Docker production incident.

## Deliverables

- container startup failure runbook
- network failure runbook
- volume/data loss runbook
- image pull/registry runbook
- high CPU/memory/OOM runbook
- CI/CD deployment failure runbook

## Acceptance Criteria

- each runbook starts with scope and user impact
- each runbook lists evidence commands before mitigations
- each runbook has safe and unsafe actions
- each runbook ends with prevention items

## Interview Proof

```text
I can troubleshoot Docker incidents using evidence, not random restarts.
```
