# Docker Interview Question Bank

## Beginner

1. What is Docker?
2. What is an image?
3. What is a container?
4. What is a Dockerfile?
5. What is build context?
6. Why use `.dockerignore`?

## Intermediate

1. How do layers and cache work?
2. What is a multi-stage build?
3. How do ports work in Docker?
4. What is the difference between volume and bind mount?
5. How does Compose service DNS work?
6. How do you debug a container that exits?

## Senior

1. How do cgroups relate to Docker resource limits?
2. What is the risk of running containers as root?
3. Why is `--privileged` dangerous?
4. How do you prevent secrets from entering image layers?
5. What is the difference between image tag and digest?
6. How would you design Docker image promotion in CI/CD?
7. How do you debug a production container incident?
8. When is Compose enough, and when do you need Kubernetes or another orchestrator?
