# MongoDB System Design Prompts

Use this format for every answer:

1. Requirements and access patterns.
2. Collections and schema.
3. Indexes.
4. Queries and aggregations.
5. Consistency model.
6. Replication/sharding strategy.
7. Failure scenarios.
8. Monitoring and operations.

## Prompts

1. Design an e-commerce product catalog.
2. Design a shopping cart service.
3. Design order management with payment and inventory.
4. Design chat message storage.
5. Design a notification system.
6. Design an audit log platform.
7. Design IoT telemetry ingestion and dashboards.
8. Design an event store.
9. Design product search with Atlas Search or Elasticsearch.
10. Design a GenAI RAG document store.
11. Design multi-tenant SaaS using MongoDB.
12. Design a real-time analytics dashboard.
13. Design a social feed backend.
14. Design a globally scalable order system using MongoDB sharding.
15. Design a cache invalidation pipeline using change streams.

## Interview Rule

Do not only list collections. A strong answer explains why each document shape fits reads, writes, growth, indexes, consistency, and failure.
