# Pinecone and VectorDB Mastery Sheet System - Start Here

This folder is a modular beginner-to-pro vector database track.

Start with:

- [Pinecone-VectorDB-Interview-Track-Index.md](Pinecone-VectorDB-Interview-Track-Index.md)
- [vector-db-mastery-lab/README.md](vector-db-mastery-lab/README.md)
- [vector-db-mastery-lab/LEARNING_PATH.md](vector-db-mastery-lab/LEARNING_PATH.md)

The learning choice:

```text
Pinecone = managed production reference
Qdrant = runnable local lab engine
VectorDB mastery = vendor-neutral retrieval architecture
```

The modular track has:

- `01-Starter-Path` for vector DB fundamentals, embeddings, similarity metrics, Pinecone/Qdrant setup, CRUD, namespaces, and metadata.
- `02-Intermediate-Backend` for ANN indexes, chunking/schema, APIs, filtering, hybrid search, RAG retrieval, and evaluation.
- `03-Senior-MAANG` for scaling, capacity, cost, multi-tenancy, ACL security, freshness, reindexing, observability, reranking, Pinecone managed production, and vendor architecture.
- `04-Scenario-Practice` for RAG, semantic product search, recommendations, support chatbots, fraud/risk similarity, multimodal search, system design, and database tradeoff analysis.
- `05-Special-Interview-Rounds` for anti-patterns, internals, debugging, and interview Q&A.
- `06-Practice-Upgrade` for active recall, runnable labs, mini projects, cheat sheets, roadmap, golden rules, and pro design review.
- `vector-db-mastery-lab` for local Docker/Qdrant practice, sample data, scripts, projects, runbooks, and interview prep.

Core mental model:

```text
VectorDB mastery = embedding quality + record schema + ANN index + metadata filters + reranking + evaluation + freshness + security + cost
```

Use the root index as the source of truth for study order.
